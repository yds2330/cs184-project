A custom Iris/Optifine shader pack built from scratch: deferred shadow mapping, screen-space reflections, and animated water — running live in Minecraft Java Edition.

00 — Abstract

Abstract

We built a custom Minecraft Java Edition shader pack using the Iris/Optifine shader API, transforming a vanilla, flat-lit scene into one with directional sun shadows, screen-space reflections on reflective materials (glass, ice, metal and gem blocks, and water), and animated water whose surface normals are driven by a procedural sum-of-sines height field. Our goal was to apply core real-time graphics concepts from CS184 — deferred rendering, shadow mapping, screen-space ray marching, and the relationship between height fields and surface normals — to a context that millions of people engage with daily. Starting from near-empty stub shaders (gbuffer passes that output albedo × lightmap, a shadow pass that writes constant white, and a composite pass that does nothing), we built every shading effect ourselves across eight GLSL shader files. The result runs at interactive frame rates on consumer GPU hardware inside Minecraft Java Edition.
01 — Technical Approach

Technical Approach

Iris/Optifine enforces a deferred rendering pipeline structure — fixed stage names, fixed texture slots, fixed uniform names — but every line of shading logic inside each stage is authored entirely by us. The pipeline as we built it looks like this: 

gbuffers_terrain.vsh/fsh  ─┐
                            ├──▶  colortex0   // albedo × lightmap  (lit scene color)
gbuffers_water.vsh/fsh    ─┤
                            └──▶  colortex1   // view-space normal (rgb) + reflectivity (a)  ← added by us
                                      │
                                      ▼
                            composite.vsh/fsh  ──▶  final.vsh/fsh  ──▶  screen
                                      ▲
              shadow.vsh/fsh  ──▶  shadowtex0   // 1024×1024 depth from the sun's POV
Feature 01

Directional Shadow Mapping

Sun shadows via a 1024×1024 orthographic shadow map reprojected through the depth buffer. Occluded pixels darkened 60%.

Feature 02

Material Tagging

Per-block reflectivity assigned at the vertex stage using Iris's block.properties + mc_Entity.x.

Feature 03

Screen-Space Reflections

32-step view-space ray march with Schlick Fresnel modulation. Runs only on pixels marked reflective.

Feature 04

Augmented G-Buffer

Both gbuffer stages write colortex1 packing view-space normal (rgb) and reflectivity (a) via /* DRAWBUFFERS:01 */.

Feature 05

Animated Water

Sum-of-sines height field with gradient-derived normals via finite differences, evaluated in world space.

1 · Directional Sun Shadows

The shadow pass is configured in shaders.properties: 

shadow.enabled=true
shadow.resolution=1024
shadow.buffers=shadowtex0

shadow.vsh transforms geometry using ftransform(), which applies shadowProjection × shadowModelView — an orthographic projection from the sun's point of view provided by Iris. shadow.fsh simply outputs gl_FragColor = vec4(1.0), which lets the driver write depth into shadowtex0. These two files together produce a 1024×1024 depth texture encoding the sun's view of the scene.

In composite.fsh, every on-screen pixel (excluding sky pixels where depth == 1.0) is processed as follows. We first reconstruct its view-space position from the depth buffer using the inverse projection matrix: 

// composite.fsh — position reconstruction (also reused for SSR)
vec3 viewPosFromDepth(vec2 uv, float depth) {
    vec4 ndcPos  = vec4(uv * 2.0 - 1.0, depth * 2.0 - 1.0, 1.0);
    vec4 viewPos = gbufferProjectionInverse * ndcPos;
    return viewPos.xyz / viewPos.w;
}

vec3 viewPos  = viewPosFromDepth(texcoord, depth);
vec4 worldPos = gbufferModelViewInverse * vec4(viewPos, 1.0);

We then transform worldPos into the sun's clip space, convert to NDC, remap to [0,1] UV, and compare the stored shadow depth against the pixel's depth from the sun. A constant bias of 0.005 in NDC depth units prevents shadow acne — the artefact that appears when a flat surface compares against a quantized version of its own depth. Occluded pixels receive a 60% darkening: 

// composite.fsh — shadow comparison
vec4 shadowClipPos = shadowProjection * (shadowModelView * worldPos);
vec3 shadowNDC     = shadowClipPos.xyz / shadowClipPos.w;
vec3 shadowUV      = shadowNDC * 0.5 + 0.5;

float bias        = 0.005;
float shadowDepth = texture2D(shadow, shadowUV.xy).r;
float inShadow    = (shadowUV.z - bias > shadowDepth) ? 1.0 : 0.0;
float shadowFactor = 1.0 - (inShadow * 0.6);
vec3  shadedColor  = color.rgb * shadowFactor;

2 · Material Tagging via block.properties and mc_Entity

Vanilla Minecraft's geometry stream carries no material identity — every vertex looks the same to the shader regardless of block type. Iris solves this with block.properties, a pack-level file that maps block name strings to integer IDs which Iris then exposes per-vertex as mc_Entity.x. Our block.properties declares three material classes: 

# block.properties
# ID 100 — smooth, semi-mirror surfaces
block.100 = minecraft:glass  minecraft:glass_pane
            minecraft:white_stained_glass  minecraft:orange_stained_glass
            ... (all 16 stained glass colours) ...
            minecraft:ice  minecraft:packed_ice  minecraft:blue_ice  minecraft:frosted_ice

# ID 101 — polished metal / gem blocks
block.101 = minecraft:iron_block  minecraft:gold_block  minecraft:diamond_block
            minecraft:emerald_block  minecraft:netherite_block

# ID 102 — water (gated separately so wave normals don't apply to stained glass)
block.102 = minecraft:water  minecraft:flowing_water

In gbuffers_terrain.vsh, the ID is read at the vertex stage and used to assign a reflectivity varying, forwarded to the fragment shader and ultimately packed into colortex1.a: 

// gbuffers_terrain.vsh
attribute vec4 mc_Entity;   // mc_Entity.x = integer block ID from block.properties
varying float reflectivity;

int id = int(mc_Entity.x + 0.5);
if      (id == 100) { reflectivity = 0.7; }  // glass / ice — strong, near-mirror
else if (id == 101) { reflectivity = 0.5; }  // metal & gem blocks — moderate
else                { reflectivity = 0.0; }  // everything else: matte

Water is handled in gbuffers_water.vsh, which always sets reflectivity = 0.6 for all fragments in that stage (water uses a dedicated wave-normal path and a fixed reflectivity value).

3 · Augmented G-Buffer (colortex1)

Both gbuffers_terrain.fsh and gbuffers_water.fsh declare /* DRAWBUFFERS:01 */, activating a second render target. colortex0 holds the lit scene color; colortex1 packs the view-space normal and per-pixel reflectivity: 

// gbuffers_terrain.fsh  (and gbuffers_water.fsh — same pattern)
/* DRAWBUFFERS:01 */
void main() {
    vec4 albedo = texture2D(gtexture, texcoord) * color;
    vec3 light  = texture2D(lightmap, lmcoord).rgb;

    gl_FragData[0] = vec4(albedo.rgb * light, albedo.a);          // colortex0
    gl_FragData[1] = vec4(viewNormal * 0.5 + 0.5, reflectivity);  // colortex1
}

The view-space normal viewNormal is computed in each vertex shader as normalize(gl_NormalMatrix * gl_Normal). gl_NormalMatrix is the inverse-transpose of the upper 3×3 of the modelview matrix — the correct transform for direction vectors under arbitrary linear deformations. The * 0.5 + 0.5 remap stores the signed normal in the [0,1] range of the RGBA8 backing texture; the inverse remap (* 2.0 − 1.0) is applied when reading in composite.fsh.

4 · Screen-Space Reflections (SSR)

In composite.fsh, after computing the shadow-modulated scene color, we read colortex1 and, for every pixel with reflectivity > 0.01, run a 32-step view-space ray march: 

// composite.fsh — SSR
vec4  normalRefl  = texture2D(colortex1, texcoord);
float reflectivity = normalRefl.a;
vec3  finalColor  = shadedColor;

if (reflectivity > 0.01) {
    vec3 normal     = normalize(normalRefl.rgb * 2.0 - 1.0);
    vec3 viewDir    = normalize(viewPos);        // camera → surface
    vec3 reflectDir = reflect(viewDir, normal);  // R = V − 2(V·N)N

    // Offset start to avoid self-intersection on step 1
    vec3 rayPos  = viewPos + normal * 0.05;
    vec3 rayStep = reflectDir * 0.3;             // 0.3 view-space units per step
    vec3 reflColor = vec3(0.0);
    float hit = 0.0;

    for (int i = 0; i < 32; i++) {
        rayPos += rayStep;

        vec4 clip = gbufferProjection * vec4(rayPos, 1.0);
        vec2 uv   = (clip.xy / clip.w) * 0.5 + 0.5;
        if (uv.x < 0.0 || uv.x > 1.0 || uv.y < 0.0 || uv.y > 1.0) break;

        float sampledDepth = texture2D(depthtex0, uv).r;
        vec3  sceneViewPos = viewPosFromDepth(uv, sampledDepth);

        // View space goes down −Z: "ray behind geometry" = rayPos.z < scene.z
        float diff = rayPos.z - sceneViewPos.z;
        if (diff < 0.0 && diff > -0.6) {
            reflColor = texture2D(gcolor, uv).rgb;
            hit = 1.0; break;
        }
    }

    // Schlick-style Fresnel: F = (1 − cosθ)^5
    float cosTheta    = max(dot(-viewDir, normal), 0.0);
    float fresnel     = pow(1.0 - cosTheta, 5.0);
    // Base reflectivity 40% at normal incidence, full reflectivity at grazing angles
    float reflStrength = mix(reflectivity * 0.4, reflectivity, fresnel);

    finalColor = mix(shadedColor, reflColor, reflStrength * hit);
}

The Schlick approximation (Schlick 1994) ensures reflections strengthen at grazing angles, matching real dielectric and metallic surface behaviour. SSR's fundamental trade-off — it can only reflect geometry already on screen — is accepted; a sky or cubemap fallback would be the natural extension.

5 · Animated Water with Procedural Wave Normals

Water with a flat normal produces a static, mirror-like reflection. To create natural rippling, we model the water surface as a height field h(x, z, t) defined by two traveling sine waves, then derive a time-varying surface normal from its spatial gradient. Both functions live in gbuffers_water.fsh: 

// gbuffers_water.fsh — wave model
uniform float frameTimeCounter;  // seconds since shader load, provided by Iris

float wave(vec2 p, vec2 dir, float wavelength, float speed, float amp, float t) {
    float k = 6.2831853 / wavelength;        // k = 2π / λ
    return amp * sin(dot(p, dir) * k + speed * t);
}

float heightField(vec2 p, float t) {
    return wave(p, normalize(vec2( 1.0,  0.6)), 4.0, 1.5, 0.04,  t)  // wave 1
         + wave(p, normalize(vec2(-0.7,  1.0)), 2.5, 2.1, 0.025, t); // wave 2
}

The surface normal is derived from the gradient of h using forward finite differences with ε = 0.05. The unit normal of the surface y = h(x,z) is n = normalize(−∂h/∂x, 1, −∂h/∂z) — derived from the cross product of the two natural surface tangents: 

// gbuffers_water.fsh — normal from height field (water fragments only)
if (isWater > 0.5) {
    vec2  p   = worldPos.xz;       // true world XZ — not camera-relative
    float t   = frameTimeCounter;
    float eps = 0.05;

    float hC = heightField(p,                  t);
    float hX = heightField(p + vec2(eps, 0.0), t);
    float hZ = heightField(p + vec2(0.0, eps), t);

    vec3 worldNormal = normalize(vec3(-(hX - hC) / eps, 1.0, -(hZ - hC) / eps));

    // Rotate world-space normal into view space for colortex1
    outNormalView = normalize((gbufferModelView * vec4(worldNormal, 0.0)).xyz);
}

The worldPos variable is reconstructed in gbuffers_water.vsh by undoing the view transform and adding the camera world offset: 

// gbuffers_water.vsh — world position
vec4 viewVertex = gl_ModelViewMatrix * gl_Vertex;
worldPos = (gbufferModelViewInverse * viewVertex).xyz + cameraPosition;
isWater  = (int(mc_Entity.x + 0.5) == 102) ? 1.0 : 0.0;

Without adding cameraPosition, the wave field evaluates in camera-relative coordinates and ripples scroll as the player moves. The isWater flag is essential because Iris routes all translucent geometry — including stained glass — through gbuffers_water. Without the gate, stained glass blocks ripple like water.

The final.vsh/fsh stage simply blits gcolor (the output of composite) to screen — no additional processing needed: 

// final.fsh
void main() {
    gl_FragColor = texture2D(gcolor, texcoord);
}
02 — Problems & Lessons

Problems Encountered

ProblemHow We Solved It
Distinguishing block materials Vanilla Minecraft's geometry stream carries no material identity — every vertex is indistinguishable to the shader. We discovered Iris's block.properties + mc_Entity.x mechanism, which lets us declare integer material class IDs at the pack level and read them in gbuffers_terrain.vsh and gbuffers_water.vsh.
View-space Z sign (SSR) OpenGL view space looks down −Z, so "the ray passed behind visible geometry" means rayPos.z < sceneViewPos.z, not the reverse. Our first SSR implementation had this backwards, producing reflections that faced the wrong direction entirely. We traced through the math on paper and corrected the comparison to diff = rayPos.z − sceneViewPos.z; if (diff < 0.0 && diff > −0.6).
SSR self-intersection noise The initial ray march started at exactly viewPos. Step 1 immediately detected the surface it was launched from, producing a mottled noisy self-reflection across the entire reflective surface. Offsetting the ray origin by normal * 0.05 before the loop resolved this completely.
Stained glass rippling like water Iris routes all translucent geometry through gbuffers_water.vsh/fsh, not just water. Our first wave-normal implementation applied the perturbation to all fragments in that stage — causing stained glass blocks to visibly ripple. We added the isWater varying (set to 1.0 only when mc_Entity.x == 102 in the vertex shader) and gated the wave computation on that flag in the fragment shader.
Wave ripples sliding with the camera Iris stores vertex positions in camera-relative coordinates for floating-point precision. Evaluating the wave field in those coordinates caused ripples to scroll across the water surface as the player moved. Adding cameraPosition (a uniform provided by Iris) to the view-inverse position produced the absolute world coordinates needed for a world-anchored wave field.
Shadow acne on flat surfaces Without a depth bias, flat surfaces compared their reprojected depth against a discretized shadow map value from the same surface, producing alternating lit and shadowed bands. A constant bias of 0.005 in NDC depth units was sufficient to push comparisons past the quantization threshold on all tested surfaces.

Lessons Learned

Deferred shading is a general scaffolding

The geometry-pass → screen-space-pass pattern hosts shadows, SSR, and water normals from the same G-buffer. SSAO, bloom, and volumetrics would all fit in the same framework.

Screen-space techniques are a pragmatic trade-off

SSR is fast and convincing on-screen but cannot reflect off-camera geometry. Knowing and accepting that limit — rather than chasing a perfect solution — is part of real-time graphics.

Height fields → normals via the gradient

The derivation n = normalize(−∂h/∂x, 1, −∂h/∂z) comes from the cross product of surface tangents. Working through it from first principles made the math genuinely ours.

Coordinate frame discipline is everything

Most debugging was not algorithmic — it was tracking which frame (object, world, view, clip, NDC) each vector lived in at each pipeline stage, and ensuring every matrix transform matched.
