00 — Abstract

Abstract

We built a custom Minecraft Java Edition shader pack using the Iris/Optifine shader API, transforming a vanilla, flat-lit scene into one with directional sun shadows, screen-space reflections on reflective materials (glass, ice, metal and gem blocks, and water), and animated water whose surface normals are driven by a procedural sum-of-sines height field. Our goal was to apply core real-time graphics concepts from CS184 — deferred rendering, shadow mapping, screen-space ray marching, and the relationship between height fields and surface normals — to a context that millions of people engage with daily. Starting from near-empty stub shaders (gbuffer passes that output albedo × lightmap, a shadow pass that writes constant white, and a composite pass that does nothing), we built every shading effect ourselves across eight GLSL shader files. The result runs at interactive frame rates on consumer GPU hardware inside Minecraft Java Edition.
01 — Technical Approach

Technical Approach

Iris/Optifine enforces a deferred rendering pipeline structure — fixed stage names, fixed texture slots, fixed uniform names — but every line of shading logic inside each stage is authored entirely by us. The pipeline as we built it looks like this: 

gbuffers_terrain.vsh/fsh  ─┐
                            ├──▶  colortex0   // albedo × lightmap  (lit scene color)
gbuffers_water.vsh/fsh    ─┤
                            └──▶  colortex1   // view-space normal (rgb) + reflectivity (a)  ← added by us
                                      │
                                      ▼
                            composite.vsh/fsh  ──▶  final.vsh/fsh  ──▶  screen
                                      ▲
              shadow.vsh/fsh  ──▶  shadowtex0   // 1024×1024 depth from the sun's POV
Feature 01

Directional Shadow Mapping

Sun shadows via a 1024×1024 orthographic shadow map reprojected through the depth buffer. Occluded pixels darkened 60%.

Feature 02

Material Tagging

Per-block reflectivity assigned at the vertex stage using Iris's block.properties + mc_Entity.x.

Feature 03

Screen-Space Reflections

32-step view-space ray march with Schlick Fresnel modulation. Runs only on pixels marked reflective.

Feature 04

Animated Water

Two-wave sum-of-sines height field, gradient-derived normals via finite differences, anchored in world space.

1 · Directional Sun Shadows

The shadow pass is configured in shaders.properties:

shadow.enabled=true
shadow.resolution=1024
shadow.buffers=shadowtex0

shadow.vsh transforms geometry using ftransform(), which applies shadowProjection × shadowModelView — an orthographic projection from the sun's point of view provided by Iris. shadow.fsh simply outputs gl_FragColor = vec4(1.0), which lets the driver write depth into shadowtex0. These two files together produce a 1024×1024 depth texture encoding the sun's view of the scene.

In composite.fsh, every on-screen pixel (excluding sky pixels where depth == 1.0) has its view-space position reconstructed from the depth buffer, then transformed into the sun's clip space for depth comparison. A bias of 0.005 prevents shadow acne. Occluded pixels are darkened by 60%: 

// composite.fsh — position reconstruction
vec3 viewPosFromDepth(vec2 uv, float depth) {
    vec4 ndcPos  = vec4(uv * 2.0 - 1.0, depth * 2.0 - 1.0, 1.0);
    vec4 viewPos = gbufferProjectionInverse * ndcPos;
    return viewPos.xyz / viewPos.w;
}

vec3 viewPos  = viewPosFromDepth(texcoord, depth);
vec4 worldPos = gbufferModelViewInverse * vec4(viewPos, 1.0);

// Shadow comparison
vec4 shadowClipPos = shadowProjection * (shadowModelView * worldPos);
vec3 shadowUV      = shadowClipPos.xyz / shadowClipPos.w * 0.5 + 0.5;
float inShadow     = (shadowUV.z - 0.005 > texture2D(shadow, shadowUV.xy).r) ? 1.0 : 0.0;
float shadowFactor = 1.0 - inShadow * 0.6;
vec3  shadedColor  = color.rgb * shadowFactor;

2 · Material Tagging via block.properties and mc_Entity

Vanilla Minecraft's geometry stream carries no material identity. Iris's block.properties maps block names to integer IDs exposed per-vertex as mc_Entity.x. We declare three material classes: 

# block.properties
block.100 = minecraft:glass  minecraft:glass_pane
            minecraft:white_stained_glass ... (all 16 colours)
            minecraft:ice  minecraft:packed_ice  minecraft:blue_ice  minecraft:frosted_ice

block.101 = minecraft:iron_block  minecraft:gold_block  minecraft:diamond_block
            minecraft:emerald_block  minecraft:netherite_block

block.102 = minecraft:water  minecraft:flowing_water

In gbuffers_terrain.vsh, the ID assigns a reflectivity varying:

// gbuffers_terrain.vsh
int id = int(mc_Entity.x + 0.5);
if      (id == 100) { reflectivity = 0.7; }  // glass / ice
else if (id == 101) { reflectivity = 0.5; }  // metal & gem blocks
else                { reflectivity = 0.0; }  // everything else: matte

Water is handled in gbuffers_water.vsh with a fixed reflectivity = 0.6.

3 · Augmented G-Buffer (colortex1)

Both gbuffer stages declare /* DRAWBUFFERS:01 */, activating a second render target. colortex1 packs the view-space normal (remapped to [0,1]) and per-pixel reflectivity: 

/* DRAWBUFFERS:01 */
void main() {
    vec4 albedo = texture2D(gtexture, texcoord) * color;
    vec3 light  = texture2D(lightmap, lmcoord).rgb;
    gl_FragData[0] = vec4(albedo.rgb * light, albedo.a);          // colortex0
    gl_FragData[1] = vec4(viewNormal * 0.5 + 0.5, reflectivity);  // colortex1
}

The view-space normal is normalize(gl_NormalMatrix * gl_Normal) — the inverse-transpose of the modelview upper 3×3, correct for direction vectors under arbitrary linear deformations.

4 · Screen-Space Reflections (SSR)

For every pixel with reflectivity > 0.01, we reflect the view direction about the surface normal and march 32 steps of 0.3 view-space units. The ray start is offset by normal * 0.05 to avoid self-intersection. On hit, we apply a Schlick Fresnel blend: 

// composite.fsh — SSR
vec3 normal     = normalize(normalRefl.rgb * 2.0 - 1.0);
vec3 viewDir    = normalize(viewPos);
vec3 reflectDir = reflect(viewDir, normal);   // R = V − 2(V·N)N

vec3 rayPos  = viewPos + normal * 0.05;
vec3 rayStep = reflectDir * 0.3;
float hit = 0.0; vec3 reflColor = vec3(0.0);

for (int i = 0; i < 32; i++) {
    rayPos += rayStep;
    vec4 clip = gbufferProjection * vec4(rayPos, 1.0);
    vec2 uv   = (clip.xy / clip.w) * 0.5 + 0.5;
    if (uv.x < 0.0 || uv.x > 1.0 || uv.y < 0.0 || uv.y > 1.0) break;
    vec3 sceneViewPos = viewPosFromDepth(uv, texture2D(depthtex0, uv).r);
    float diff = rayPos.z - sceneViewPos.z;
    if (diff < 0.0 && diff > -0.6) { reflColor = texture2D(gcolor, uv).rgb; hit = 1.0; break; }
}

// Schlick Fresnel: F = (1 − cosθ)^5
float fresnel     = pow(1.0 - max(dot(-viewDir, normal), 0.0), 5.0);
float reflStrength = mix(reflectivity * 0.4, reflectivity, fresnel);
finalColor = mix(shadedColor, reflColor, reflStrength * hit);

5 · Animated Water with Procedural Wave Normals

We model the water surface as a height field h(x,z,t) — a sum of two traveling sine waves — and derive a time-varying surface normal from its gradient: 

// gbuffers_water.fsh
float wave(vec2 p, vec2 dir, float wavelength, float speed, float amp, float t) {
    float k = 6.2831853 / wavelength;
    return amp * sin(dot(p, dir) * k + speed * t);
}

float heightField(vec2 p, float t) {
    return wave(p, normalize(vec2( 1.0,  0.6)), 4.0, 1.5, 0.04,  t)
         + wave(p, normalize(vec2(-0.7,  1.0)), 2.5, 2.1, 0.025, t);
}

// Normal from gradient via finite differences (ε = 0.05)
if (isWater > 0.5) {
    float hC = heightField(worldPos.xz, frameTimeCounter);
    float hX = heightField(worldPos.xz + vec2(0.05, 0.0), frameTimeCounter);
    float hZ = heightField(worldPos.xz + vec2(0.0, 0.05), frameTimeCounter);
    vec3 worldNormal = normalize(vec3(-(hX-hC)/0.05, 1.0, -(hZ-hC)/0.05));
    outNormalView = normalize((gbufferModelView * vec4(worldNormal, 0.0)).xyz);
}

World position is reconstructed in gbuffers_water.vsh as (gbufferModelViewInverse * viewVertex).xyz + cameraPosition so ripples don't drift with the camera. The isWater flag (from mc_Entity.x == 102) gates wave normals away from stained glass, which shares this shader stage.

02 — Problems & Lessons

Problems Encountered

ProblemHow We Solved It
Distinguishing block materials Vanilla geometry carries no material identity. We used Iris's block.properties + mc_Entity.x to declare integer material class IDs readable in the vertex shader.
View-space Z sign (SSR) OpenGL view space goes down −Z, so "ray behind geometry" means rayPos.z < scenePos.z. Our first implementation had this backwards. Fixed after tracing through the math on paper.
SSR self-intersection noise Ray starting at exactly viewPos immediately hit the surface it launched from. Fixed by offsetting the ray origin by normal * 0.05.
Stained glass rippling like water All translucents share gbuffers_water. We added an isWater varying gated on mc_Entity.x == 102 to restrict wave normals to actual water.
Wave ripples sliding with the camera Iris stores positions camera-relative. Adding cameraPosition to the view-inverse position anchored the wave field in world space.
Shadow acne on flat surfaces A constant depth bias of 0.005 in NDC units eliminated quantization artefacts on all tested surfaces.
